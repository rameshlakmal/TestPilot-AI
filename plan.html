Test Scenario Generator — Project Plan

An AI-powered Next.js application that transforms software requirements into structured, high-quality test scenarios using multiple test design techniques and LLM providers.

What the Project Achieves

Core Goal: Accept raw software requirements (text or documents), use AI to analyze them, recommend appropriate test design techniques, and then generate comprehensive, structured test scenarios — ready for export or direct push to test management tools like Jira/Xray.

For QA Engineers

Eliminates hours of manual test case writing. Paste a requirement, get dozens of well-structured scenarios in seconds.

For Test Leads

Ensures consistent test coverage across the team using proven techniques like Boundary Value Analysis and Decision Tables.

For Teams

Export to CSV/JSON/PDF or push directly to Jira Xray. Integrates into existing workflows without friction.

High-Level Workflow

The app follows a 3-step wizard flow managed by a Zustand store with session persistence.

Step 1 — Input

Paste text or upload a document (PDF, DOCX, MD, TXT). Optionally split into chunks. Select AI provider.

▼

Step 2 — Analysis

AI analyzes the requirement, extracts testable elements, and recommends test techniques with confidence levels.

▼

Step 3 — Results

AI generates scenarios per technique. Deduplication runs. Review, export (CSV/JSON/PDF), or push to Jira Xray.

Detailed Workflow Breakdown

1

Requirement Input

Page: /  |  Route: app/page.tsx
  • Text Input: User types or pastes requirement text directly into a textarea.
  • File Upload: User uploads a file (PDF, DOCX, Markdown, TXT). The file is sent to /api/parse-document which uses format-specific parsers (mammoth for DOCX, pdf-parse for PDF, etc.) to extract text.
  • AI Provider Selection: User picks between OpenAI Gemini Claude.
  • Chunking (Optional): For large documents, the user can click "Split into Requirements" which calls /api/chunk. The AI breaks the document into individual, self-contained requirement chunks (REQ-001, REQ-002, etc.). The user can then select/deselect which chunks to analyze.
  • Analyze: Clicking "Analyze Requirement" sends the text (or selected chunks) + provider to /api/analyze.
2

Requirement Analysis

Page: /analysis  |  Route: app/analysis/page.tsx

The /api/analyze route sends the requirement to the chosen AI provider with a structured output schema. The AI returns:

Extracted Elements

Testable elements identified in the requirement:

input output state rule boundary constraint action

Each element has a name, type, description, and optional values.

Technique Recommendations

The AI recommends test design techniques with:

  • Confidence level (high/medium/low)
  • Rationale explaining why
  • Estimated scenario count

High/medium confidence techniques are auto-selected.

The user reviews the analysis, toggles techniques on/off, and clicks "Generate Test Scenarios" to proceed.

3

Scenario Generation & Results

Page: /results  |  Route: app/results/page.tsx

The /api/generate route fires off parallel AI calls (max concurrency: 3), one per selected technique. Each call uses a technique-specific prompt template.

Generated Scenario Structure

Each generated test scenario contains:

  • ID — Unique identifier
  • Title & Description
  • Preconditions — Setup required
  • Steps — Action + Expected Result + Test Data
  • Expected Result — Overall outcome
  • Priority — critical / high / medium / low
  • Type — positive / negative / edge-case / boundary
  • Tags — Categorization labels

Post-Generation Processing

  • Deduplication: A Jaccard similarity algorithm compares scenario titles (40%), steps (40%), and descriptions (20%). Pairs above 60% similarity are flagged as duplicates. Users can dismiss them with one click.
  • Stats Dashboard: Shows total count, priority breakdown (critical/high), scenario types (positive/negative/edge-case/boundary), and duplicate count.

Key Design Decisions

Structured Output via Zod

All AI responses are validated against Zod schemas using zod-to-json-schema to generate JSON Schema for providers. This ensures type-safe, predictable output regardless of AI hallucinations.

Technique-Specific Prompts

Each test design technique has its own dedicated prompt template with specialized system instructions, ensuring domain-expert-level output per technique.

Parallel Generation

When multiple techniques are selected, scenario generation runs in parallel (max 3 concurrent) to minimize total wait time.

Deduplication

Since multiple techniques may produce overlapping scenarios, a Jaccard-similarity algorithm flags duplicates (>60% match) for user review.

Session Persistence

The Zustand store persists to sessionStorage, so users don't lose progress on page refresh within the same session.

Provider Abstraction

A common AIProvider interface means adding a new AI provider only requires implementing one method — no changes to the rest of the application.
