Work Plan: AI Test Scenario Generator Agent

Overview

Build an AI agent that accepts software requirements or user stories and automatically generates comprehensive test scenarios using formal test design techniques, Mermaid.js diagrams, and structured test cases with test data.

Problem Statement

Manual test design is time-consuming and often incomplete. Testers need to:

  • Choose the right test design technique for each requirement
  • Ensure full coverage (positive, negative, edge cases)
  • Produce structured, readable test artifacts

This agent automates that entire process.

Goals

  1. Accept requirements as text input, uploaded file, or imported directly from Jira user stories
  2. Automatically select the most appropriate test design technique(s)
  3. Generate a formal test design with reasoning
  4. Produce Mermaid.js diagrams relevant to the technique chosen
  5. Output structured test scenarios with concrete test data
  6. Push generated test cases back to Jira/AIO test management

Test Design Techniques the Agent Will Know

Technique Best Used When
Equivalence Partitioning Inputs can be grouped into classes that behave the same way
Boundary Value Analysis Numeric ranges, limits, or thresholds are involved
Decision Table Testing Multiple conditions combine to produce different outcomes
State Transition Testing System has distinct states and transitions between them
Domain Analysis Testing Complex domains with multiple interacting variables

Agent Workflow (Pipeline)

[User Input: Text / File Upload / Jira Import]
          │
          ▼
  Step 0: Input Processing
  → Accept text, file upload, or Jira user stories
  → Jira: select project → filter by epic/sprint/status → pick stories → auto-advance
  → Jira stories stay synced with checkboxes (reactive update)
          │
          ▼
  Step 1: Requirement Analyzer
  → AI extracts testable elements: inputs, outputs, rules, states, boundaries
  → Recommends 3-6 testing techniques with confidence levels
          │
          ▼
  Step 2: Technique Selection (User Review)
  → User toggles recommended techniques on/off
  → Optionally selects Mermaid.js diagrams per technique
          │
          ▼
  Step 3: Per-Skill Parallel Generation
  → Each technique gets a dedicated, focused LLM call
  → Up to 3 concurrent calls (1 for Gemini)
  → Optional Mermaid diagram generated alongside test cases
          │
          ▼
  Step 4: Merge + Deduplicate
  → Combine results from all skills
  → Weighted Jaccard dedup (title 40%, steps 40%, expected 20%, threshold 60%)
  → Renumber IDs sequentially
          │
          ▼
  [Output: Displayed in Web UI + Export JSON/CSV + Push to Jira/AIO]

Output Artifacts

For each requirement processed, the agent produces:

Artifact Format Description
Analysis Report Markdown Technique selected + reasoning
Test Design Markdown table Partitions / boundaries / states / decisions
Mermaid.js Diagrams SVG Client-side rendered Mermaid.js diagrams
Test Cases JSON Structured test cases with steps, expected results, and concrete test data

Phase-by-Phase Build Plan

Phase 1 — Project Scaffold

  • Create folder structure (server/, client/, skills/)
  • Set up package.json and .env.example
  • Verify Express server + LLM provider connections work

Phase 2 — Agent Core (Prompts + Routes)

  • Write prompt builders for analysis, per-skill generation, and preflight
  • Build Express routes (analyze → generate-tests with per-skill parallel calls)
  • Build orchestration logic in server/index.js

Phase 3 — Mermaid.js Diagrams

  • Integrate Mermaid.js for client-side SVG rendering
  • Test all diagram types render correctly

Phase 4 — React UI

  • Build React + MUI 3-step wizard with input, analysis, and results views
  • Wire UI to the Express API endpoints
  • Add export (JSON/CSV) and push-to-Jira functionality

Phase 5 — Testing & Refinement

  • Test with sample requirements (login form, e-commerce checkout, state machine)
  • Refine prompts based on output quality
  • Add error handling and edge case management

Example Input / Output

Input (User Story):

As a user, I want to log in with my email and password. The password must be 8-20 characters. After 3 failed attempts, the account is locked for 15 minutes.

Expected Output:

  • Technique selected: Boundary Value Analysis + State Transition Testing (with reasoning)
  • Mermaid.js State Diagram showing: Unlocked → Failed Attempt → Locked → Cooldown states
  • Test Cases including:
    • Positive: valid email + valid password → login success
    • Negative: wrong password → error message shown
    • Edge: exactly 3 failed attempts → account locked
    • Boundary: password of 8 chars (valid), 7 chars (invalid), 20 chars (valid), 21 chars (invalid)
